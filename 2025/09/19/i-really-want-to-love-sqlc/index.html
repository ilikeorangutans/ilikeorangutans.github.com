[DRAFT] I Really Want to Love sqlc

go golang sqlc sql orm

I really, really like sqlc. If you’re unfamiliar, it’s a tool that takes (slightly) annotated SQL and generates Go code to execute these queries. I prefer it over ORMs as I don’t have to learn another specific DSL that always tends to break down when I have to write specific joins or other aggregations. Instead, I write raw SQL. Another nice property of the custom tailored go code is that it’s not making use of any. Instead everything is a concrete type giving you extra type safety.

Once sqlc is set up, it works really well. Except… there are some rough corners that I keep running into. The first one is dynamic queries, and that’s usually the first thing one encounters when using sqlc. We’re so used to just dynamically build queries that we expect to be able to say select * from table where $column = $value order by $order_column. sqlc can’t quite do that. You can kind of get around it.
